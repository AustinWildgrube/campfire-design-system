# Inline Notifications
Inline Notifications are used for contextual information, including positive feedback or confirmation about an action, warnings about possible issues, and
errors. They are used to provide context in close proximity to a piece of content, preferably at the top.

Campfire inline notifications are called via the `<usi-inline>` element. Inline notifications only require two properties as shown below.

```html
<usi-inline usiTitle="Lorem Ipsum" usiMessage="Lorem ipsum dolor sit amet"></usi-inline>
```

## Inline Notification Types
There are five different inline notification types: `success`, `warning`, `error`, `info`, `default`. Order inline notifications by the urgency with which the
user needs to pay attention to: error, warning, info, and then success. In the case where there are multiples of each type, all of them are shown before another
type is reached.

### Success
Reserved to provide a static, persistent success message.

```html
<usi-inline usiTitle="Lorem Ipsum" usiMessage="Lorem ipsum dolor sit amet" usiType="success"></usi-inline>
```

### Warning
Reserved for warnings: a message that needs the user's attention and acknowledgement but might not cause errors.

```html
<usi-inline usiTitle="Lorem Ipsum" usiMessage="Lorem ipsum dolor sit amet" usiType="warning"></usi-inline>
```

### Error
Reserved for errors, malfunctions, and critical issues like license expiration.

```html
<usi-inline usiTitle="Lorem Ipsum" usiMessage="Lorem ipsum dolor sit amet" usiType="error"></usi-inline>
```

### Info
Reserved for non-critical information that can be used to assist the user.

```html
<usi-inline usiTitle="Lorem Ipsum" usiMessage="Lorem ipsum dolor sit amet" usiType="info"></usi-inline>
```

### Default
Reserved for custom inline notifications.

> While using the default inline notification, it is best to use the icon property to help convey the custom message.

```html
<usi-inline usiTitle="Lorem Ipsum" usiMessage="Lorem ipsum dolor sit amet"></usi-inline>
<usi-inline usiTitle="Lorem Ipsum" usiMessage="Lorem ipsum dolor sit amet" usiIcon="alien"></usi-inline>
<usi-inline usiTitle="Lorem Ipsum" usiMessage="Lorem ipsum dolor sit amet" usiType="default"></usi-inline>
```

## Custom Inline Notification Icons
Icons can be used inside inline notifications to decorate the call-to-action with a visual indicator. Icons should be used alongside text that clearly
indicates what the interaction is expected to do.

> FontAwesome is currently the ***only*** icon library Campfire supports.

```html
<usi-inline usiTitle="Lorem Ipsum" usiMessage="Lorem ipsum dolor sit amet" usiIcon="alien"></usi-inline>
```

## Inline Notification Customization
Campfire inline notifications were made to be easily customizable. Styling an inline notification is as simple as just adding a class to the tag or
overriding our CSS variables in the root selector. The list of overridable variables is as follows:

```css
:root {
  --inline-background-color: var(--white);
  --inline-border-color: var(--black);
  --inline-title-text-color: var(--black);
  --inline-message-text-color: var(--black);

  --inline-border-color--success: var(--success);
  --inline-text-color--success: var(--success);

  --inline-border-color--warning: var(--warning);
  --inline-text-color--warning: var(--warning);

  --inline-border-color--error: var(--error);
  --inline-text-color--error: var(--error);

  --inline-border-color--info: var(--info);
  --inline-text-color--info: var(--info);
}
```

```html
<usi-inline class="custom-inline" usiTitle="Custom Inline" usiMessage="Lorem ipsum dolor sit amet" usiType="success"></usi-inline>
```
