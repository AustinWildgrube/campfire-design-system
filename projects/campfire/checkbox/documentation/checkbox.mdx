# Checkboxes
Checkboxes provide a user-friendly method for selecting one or multiple options from a list or answering a question. A label or instructions that clearly
indicates the outcome of selecting the checkbox should always follow it.

Each checkbox has two states: selected or unselected, akin to an on and off switch. For instance, if the checkbox label reads "Yes, email me," it's evident
what selecting or not selecting the checkbox will achieve.

Furthermore, checkboxes display users' selections, allowing them to deselect an option, which may be challenging with other selection methods on a form, such
as radio buttons or dropdown boxes.

```html
  <usi-checkbox>This is the label</usi-checkbox>
```

## Checkboxes in Forms
Checkboxes are frequently employed in forms. Campfire checkboxes can be used in reactive forms and template-driven forms using the methods outlined below.

```ts
checkNgModel: boolean = false;
checkFormGroup: FormGroup;

constructor(private formBuilder: FormBuilder) {
  this.checkFormGroup = this.formBuilder.group({
    checkOne: true
  });
}
```

```html
<usi-checkbox [(ngModel)]="checkNgModel">Checkbox Label</usi-checkbox>

<form [formGroup]="checkFormGroup">
  <usi-checkbox formControlName="checkOne">Checkbox One</usi-checkbox>
</form>
```

## Required Checkboxes
Adding the usiRequired attribute to the checkbox label makes the checkbox mandatory. This ensures that the user selects the checkbox before submitting the
form. This is useful when users must agree to terms of service, for example.

```html
<usi-checkbox usiRequired>This is the label</usi-checkbox>
```

## Disabled Checkboxes
Disabling checkboxes indicates to users that the option is unavailable. Users should be informed why the checkbox is disabled and how to enable it if possible.

```html
<usi-checkbox usiDisabled>This is the label</usi-checkbox>
```

## Checked Checkboxes
Checkboxes can be preselected if necessary. The value will be true, and the user will not be able to modify it.

```html
<usi-checkbox usiChecked>This is the label</usi-checkbox>
```

## Customization
Checkboxes were made to be easily customizable. Styling a checkbox is as simple as just adding a class onto the tag or
overriding our CSS variables in the root selector. The list of overridable variables are as follows:

```css
:root {
  --checkbox-border-color: var(--gray);
  --checkbox-label-text-color: var(--black);

  --checkbox-background-color--checked: var(--primary);
  --checkbox-border-color--checked: var(--primary);
  --checkbox-check-color--checked: var(--white);

  --checkbox-border-color--hover: var(--primary);
  --checkbox-border-color--checked--hover: var(--black);

  --checkbox-background-color--disabled: #dfdfe3;
  --checkbox-check-color--disabled: var(--gray);
  --checkbox-label-text-color--disabled: var(--gray);
}
```

```html
<usi-checkbox class="custom-checkbox">This is the label</usi-checkbox>
```
