import { UsiRadioModule } from '@campfire/radio';

# Radio Buttons
Radio buttons are useful for when you want users to see all available options and the list of options is small. For
mutually exclusive options, consider a checkbox or toggle switch. With radio buttons only one option can be selected at
a time. When using radio buttons it is best to keep the number of options to a maximum of six.

Radio buttons should **always** be used within a group. The group syntax is very similar to that of a solo radio button
with the exception that a group is formed using the `<usi-radio-group>` element. Usi radio buttons are called via the
`usi-radio` directive on a label. They then can have custom attributes that may be used along with it. The standard
radio button is called as:

```html
<label ngModel usi-radio>Radio Button Text</label>

<usi-radio-group>
  <label usiValue="one" usi-radio>Option One</label>
  <label usiValue="two" usi-radio>Option Two</label>
  <label usiValue="three" usi-radio>Option Three</label>
</usi-radio-group>
```

## Radio Buttons in Forms
Radio buttons will almost always be used within forms. Campfire radio buttons can be used in reactive forms as well as in
template driven forms via the methods below.

```ts
radioNgModel: string = 'two';
radioFormGroup: FormGroup;

constructor(private formBuilder: FormBuilder) {
  this.radioFormGroup = this.formBuilder.group({
    controlOne: 'one'
  });
}
```

```html
<usi-radio-group [(ngModel)]="radioNgModel">
  <label usiValue="one" usi-radio>Option One</label>
  <label usiValue="two" usi-radio>Option Two</label>
  <label usiValue="three" usi-radio>Option Three</label>
</usi-radio-group>

<form [formGroup]="radioFormGroup">
  <usi-radio-group formControlName="controlOne">
    <label usiValue="one" usi-radio>Option One</label>
    <label usiValue="two" usi-radio>Option Two</label>
    <label usiValue="three" usi-radio>Option Three</label>
  </usi-radio-group>
</form>
```

## Radio Button Values
Radio buttons are given a value by using `usiValue` attribute.

> Exclusively for **singular** radio buttons

```html
<label usiValue="one" usi-radio>Option One</label>
```

## Custom Radio Button Names
`usiName` can be used to bind all input[type="radio"] to the same group. It is usually used to let the
browser see your `usi-radio-group` as a real "group" and keep the default behavior. For example, using left/right
keyboard arrow to change your selection that in the same `usi-radio-group`. This is done by default using 5 random
alphanumeric characters, but sometimes it is beneficial to know the name before the group is loaded to the DOM.

> Exclusively for **groups**

```html
<usi-radio-group usiName="custom-name">
  <label usiValue="one" usi-radio>Option One</label>
  <label usiValue="two" usi-radio>Option Two</label>
  <label usiValue="three" usi-radio>Option Three</label>
</usi-radio-group>
```

## Vertical Radio Button Groups
Radio buttons can be placed in a vertical group by adding the `usiDirection` attribute to the `<usi-radio-group>` element.
This can occasionally make the distinction between choices that much easier to read.

> Exclusively for **groups**

```html
<usi-radio-group usiDirection="vertical">
  <label usiValue="one" usi-radio>Option One</label>
  <label usiValue="two" usi-radio>Option Two</label>
  <label usiValue="three" usi-radio>Option Three</label>
</usi-radio-group>
```

## Disabled Radio Buttons
Radio buttons may be disabled to indicate to a user that the choice set is unavailable. It should be clear to the user
why a radio button is disabled, and users should be directed how enable the choice set if that is possible.

`usiDisabled` can be called on the group if all radio buttons are to be disabled. In this event a message should be
displayed to the user as to why they are all disabled.

```html
<label ngModel usiDisabled usi-radio>Radio Button Text</label>

<usi-radio-group usiDisabled>
  <label usiValue="one" usi-radio>Option One</label>
  <label usiValue="two" usi-radio>Option Two</label>
  <label usiValue="three" usi-radio>Option Three</label>
</usi-radio-group>
```

## Radio Button Loading
When the page is being initially loaded into the DOM a radio button can take advantage of ghost loading. This feature
takes advantage of Angular object binding. For dynamic loading times the square bracket syntax is required. If a radio
button is to load indefinitely it can be called as a boolean value.

Instead of individually ghosting all radio buttons on the page the `usiGhost` attribute can be used on the group.

```ts
let isGhosting = true;

// The radio buttons will load for 5 seconds
setTimeout(() => {
  isGhosting = false;
}, 5000);
```

```html
<label ngModel [usiGhost]="isGhosting" usi-radio>Radio Button Text</label>

<usi-radio-group [usiGhost]="isGhosting">
  <label usiValue="one" usi-radio>Option One</label>
  <label usiValue="two" usi-radio>Option Two</label>
  <label usiValue="three" usi-radio>Option Three</label>
</usi-radio-group>
```

## Radio Button Customization
Usi radio buttons were made to be easily customizable. Styling a radio button is as simple as just overriding our
CSS variables in the root selector. The list of overridable variables are as follows:

```css
:root {
  // Radio buttons
  --radio-border-color: var(--gray);
  --radio-label-text-color: var(--black);

  --radio-border-color--checked: var(--primary);
  --radio-background-color--checked: var(--primary);

  --radio-border-color--checked--disabled: var(--gray);
  --radio-background-color--checked--disabled: var(--gray);

  --radio-border-color--hover: var(--primary);

  --radio-background-color--disabled: #dfdfe3;
  --radio-label-text-color--disabled: var(--gray);
}
```
