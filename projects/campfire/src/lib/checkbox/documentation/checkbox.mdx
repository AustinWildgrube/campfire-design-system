# Checkboxes
Checkboxes are an easily understandable way to indicate that users can select one or more answers to a question or items
from a list. They’re always followed by a label or instructions that clearly indicate what checking the box represents.

Each checkbox has two different states: selected or unselected, which are similar to an on and off switch. If a checkbox
label says “Yes, email me,” it’s clear what checking that box (or not checking it) will accomplish.

Checkboxes also visibly show users what’s been selected and makes it easy for them to "uncheck" an option, which can be
difficult with other selection methods on a form (such as radio buttons or dropdown boxes).

```html
  <label usi-checkbox>This is the label</label>
```

## Checkboxes in Forms
Checkboxes will almost always be used in forms. Campfire checkboxes can be used in reactive forms as well as in
template driven forms via the methods below.

```ts
checkNgModel: boolean = false;
checkFormGroup: FormGroup;

constructor(private formBuilder: FormBuilder) {
  this.checkFormGroup = this.formBuilder.group({
    checkOne: true
  });
}
```

```html
<label [(ngModel)]="checkNgModel">Checkbox Label</label>

<form [formGroup]="checkFormGroup">
  <label formControlName="checkOne">Checkbox One</label>
</form>
```

## Required Checkboxes
Checkboxes can be required by adding the `required` attribute to the checkbox label. This will ensure that the user checks
the checkbox before submitting the form. This is useful for when a user must agree to a term of service for example.

```html
<label usi-checkbox required>This is the label</label>
```

## Disabled Checkboxes
Checkboxes may be disabled to indicate to a user that the choice is unavailable. It should be clear to the user
why the checkbox is disabled, and users should be directed how enable the checkbox if that is possible.

```html
<label usi-checkbox disabled>This is the label</label>
```

## Customization
Checkboxes were made to be easily customizable. Styling a checkbox is as simple as just adding a class onto the tag or
overriding our CSS variables in the root selector. The list of overridable variables are as follows:

```css
:root {
  --checkbox-border-color: var(--gray);
  --checkbox-label-text-color: var(--black);

  --checkbox-background-color--checked: var(--primary);
  --checkbox-border-color--checked: var(--primary);
  --checkbox-check-color--checked: var(--white);

  --checkbox-border-color--hover: var(--primary);

  --checkbox-background-color--disabled: #dfdfe3;
  --checkbox-check-color--disabled: var(--gray);
  --checkbox-label-text-color--disabled: var(--gray);
}
```

```html
<label class="custom-checkbox" usi-checkbox>This is the label</label>
```
